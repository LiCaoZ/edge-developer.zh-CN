# Microsoft Edge 加载项存储开发人员策略  

## 本文档的简介和目标  

感谢你有兴趣开发 Microsoft Edge 加载项存储的扩展。  Microsoft Edge 加载项存储开发人员策略 \ (加载项应用商店开发人员策略\) 适用于你的扩展，包括通过合作伙伴中心提交扩展，以及通过 Microsoft Edge 加载项提供[][MicrosoftPartnerCenter]此类扩展。  

## 原则  

下面是一些入门准则：  

*   应在 Microsoft Edge 的扩展中提供独特且不同的值。  提供从 Microsoft Edge 加载项存储 \ (Microsoft Edge 加载项\) 下载扩展的极具吸引力) 。  
*   不得在扩展功能、谁提供扩展等方面误导我们的联合用户。  
*   不得尝试欺骗用户、系统或生态系统。  我们的 Microsoft Edge 加载项中不存在任何类型的欺诈;是否进行评级和审核操作、信用卡欺诈或其他欺诈活动。  
    
遵循 Microsoft Edge 加载项应用商店开发人员策略应有助于做出选择，以增强扩展的吸引力和受众。  

扩展对于亿万用户的体验至关重要。  我们期待体验你创建并很高兴帮助你向世界提供扩展。  

## 1. 产品策略  

### 1.1 不同函数&值;准确的表示形式  

扩展和相关元数据必须准确且清楚地反映您描述的源、功能和功能。  

#### 1.1.1 扩展必须具有一个用途  

扩展必须具有单个用途，功能范围较窄。  

#### 1.1.2 描述扩展  

扩展的所有方面都应准确描述扩展的功能、功能和任何重要限制，包括必需或受支持的输入设备。  首次运行体验期间，扩展的价值主张必须清晰明了。  您的扩展不得使用与其他扩展名称或图标类似的名称或图标，如果您无权进行该表示，则不得声明代表公司、政府机构或其他实体。  

#### 1.1.3 功能  

扩展必须功能完整。  

#### 1.1.4 搜索和发现  

搜索词不能超过七个唯一术语，并且应该与扩展相关。  

#### 1.1.5 提供适当的详细信息  

您必须提供有关扩展和扩展一览 \ (metadata\) 功能的详细信息。  你的扩展必须提供有价值且高质量的用户体验。  你的扩展还必须在 Microsoft Edge 加载项中具有活动状态。  

#### 1.1.6 稳定性和性能  

扩展不得对 Microsoft Edge 的性能或稳定性产生负面影响。  

#### 1.1.7 模糊处理  

不允许使用模糊代码的扩展。  这包括扩展包中的代码以及从 Web 获取的任何外部代码或资源。  如果代码不可审阅，系统可能会要求你重构代码的某些部分。  

#### 1.1.8 更改浏览器设置  

未经适当的用户同意，您的扩展不得更改或显示更改浏览器功能或设置，包括但不限于：地址栏搜索提供程序和建议、起始页或主页、新选项卡页以及添加或删除收藏夹。  

对浏览器设置的任何更改都应明确记录在扩展说明中。  

您的扩展可能仅修改关键设置，以将 Microsoft 网页或服务替换为第三方 \ (（例如，要求使用第三方搜索引擎）或将主页设置为第三方 Web 属性\) （如果你被此类第三方雇用或以其他方式关联）。  

### 1.2 安全性  

你的扩展不得危及或损害用户安全，或设备、系统或相关系统的安全或功能。  

#### 1.2.1 内容安全策略  

> [!NOTE]
> 如果对扩展进行了超出所述功能的任何更改，则对代码的任何更改都必须符合 [Microsoft Edge][MicrosoftEdgeContentSecurityPolicyRemoteScript]内容安全策略。  例如，您的扩展不应下载远程脚本，并随后以与描述的功能不一致的方式运行该脚本。  

#### 1.2.2 不需要和恶意软件  

扩展不得包含或启用 Microsoft 针对不需要和恶意软件的条件定义的 [恶意软件][MicrosoftIdentifiesMalwareUnwantedApplications]。  

#### 1.2.3 依赖其他软件  

扩展可能依赖于非集成软件 \ (例如其他产品、模块或服务\) 以提供主要功能，但您必须在说明中披露依赖项  

#### 1.2.4 扩展更新  

除非 Microsoft 另行允许，否则只能通过 Microsoft Edge 加载项更新扩展。  

### 1.3 产品可测试  

扩展必须可测试。  如果出于任何原因（包括但不限于以下项目）无法测试扩展，您的扩展可能无法满足此要求。  

#### 1.3.1 用户凭据  

如果您的扩展需要登录凭据，则使用该字段提供一个工作演示 `Notes for certification` 帐户。  

#### 1.3.2 服务的可用性  

如果您的扩展需要访问服务器，则服务器必须正常工作，以验证其是否正常工作。  

### 1.4 可用性  

扩展必须符合 Microsoft Edge 加载项存储可用性标准，包括但不限于以下子部分中列出的标准。  

#### 1.4.1 跨平台的兼容性  

扩展应在可下载它们的所有设备和平台上与 Microsoft Edge 兼容。  如果在与扩展不兼容的设备上下载扩展，它应在启动时检测该扩展，并向用户显示一条消息，详细说明设备必须满足的要求，以便与扩展兼容。  

#### 1.4.2 用户体验  

扩展必须迅速启动，并且必须保持对用户输入的响应。  扩展必须继续运行，并始终响应用户输入。  扩展必须正常关闭，且不能意外关闭。  处理异常后，扩展应处理异常，并始终响应用户输入。  

### 1.5 个人信息  

以下要求适用于访问个人信息的扩展。  个人信息包括标识或可用于标识人员或与此类信息或数据关联的所有信息或数据。  

#### 1.5.1 仅在必要时收集个人信息  

您的扩展可能会收集、访问、使用或传输个人信息 \ (包括 Web 浏览活动\) ;仅在需要且仅在公开公开、面向用户的功能中使用时。  

#### 1.5.2 维护隐私策略  

无论您的扩展是访问、收集还是传输个人信息;如果法律要求，你必须提供醒目的通知并遵守隐私策略。  隐私策略必须通知用户扩展访问、收集或传输的个人信息、该信息的使用、存储和保护方式，并指示披露该信息的各方类型。  隐私策略必须描述用户对信息的使用和共享的控制、他们如何访问其信息，并且必须遵守适用的法律和法规。  向扩展添加新特性和功能时，隐私策略必须保持最新。  

如果你向 Microsoft 提供隐私策略，即表示你同意允许 Microsoft 与你扩展的用户共享此类隐私策略。  

#### 1.5.3 与第三方共享数据  

只有在获得这些用户的选择加入同意后，才能通过扩展或关联元数据将扩展用户的个人信息发布到外部服务或第三方。  选择加入同意意味着用户在您执行以下操作后，在扩展的用户界面中为所请求的活动提供其明确权限：  

*   向用户介绍如何访问、使用或共享信息，并指示披露该信息的各方类型，  
*   在扩展用户界面中为用户提供一种机制，他们可以通过该机制选择稍后取消权限和选择退出。  
    
#### 1.5.4 非用户的共享信息  

如果您通过扩展或元数据将某人的个人信息发布到外部服务或第三方，但共享其信息的用户不是您的扩展用户;  

1.  您必须获得明确书面同意才能发布该个人信息。  
1.  必须允许共享其信息的人随时撤销该同意。  
1.  你的隐私策略必须明确披露你可能会这样收集个人信息。  
1.  如果适用法律要求，您必须在请求时删除任何个人的个人信息，包括你通过此方式收集的信息的个人。  
1.  如果您的扩展为用户提供了对其他人个人信息的访问权限，则此要求也适用。  
    
#### 1.5.5 安全地传输信息  

如果您的扩展收集、存储或传输个人信息;必须使用新式加密方法安全执行。  

#### 1.5.6 高度敏感信息  

您的扩展不得收集、存储或传输高度敏感的个人信息，如健康或财务数据，除非该信息与扩展的功能相关。  在收集、存储或传输此类信息之前，扩展还必须征得用户明确同意。  

### 1.6 权限  

扩展必须仅请求正常运行所需的权限。  您必须提供扩展工作原理的说明。  扩展必须仅执行所述的操作。  扩展可能不会请求超出声明的执行和运行所需功能的权限。  

### 1.7 本地化  

应为扩展声明支持的所有语言本地化扩展。  扩展说明的文本应该以您声明的每种语言进行本地化。  
如果您的扩展已本地化，因此某些功能在本地化版本中不可用，则必须在扩展说明中清楚地说明或显示本地化限制。 扩展提供的体验在它支持的所有语言中必须相当类似。  

### 1.8 财务事务  

如果你的产品包括产品内购买、订阅、虚拟货币、计费功能或捕获财务信息;以下各节中的要求适用。  

#### 1.8.1 付费功能  

你的扩展可能允许用户使用通过第三方购买机制或 API 购买的数字内容或服务。  

必须使用安全的第三方购买 API 购买物理商品或服务。  你必须使用安全的第三方购买 API，用于与任何其他服务（包括实际购物或资金贡献）有关的付款。  

*   如果你的扩展用于促进或收集社会性贡献，或进行促销性宣传活动或比赛，则必须遵循适用法律。  
*   您还必须明确声明，Microsoft 不是此次促销活动的集资者或主办方。  
*   在扩展中出售的产品内产品/服务不得转换为任何合法有效的货币 \ (如美元、欧元等\) 或任何物理商品或服务。  
    
以下要求适用于使用安全第三方购买 API：  

*   在交易时或收集用户的任何付款或财务信息时;您的扩展必须标识商业交易提供商、对用户进行身份验证，并获取交易的用户确认。  商业交易提供商维护一个安全的金融交换平台。  
*   您的扩展可能为用户提供保存此身份验证的能力，但用户必须能够要求每次事务进行身份验证或关闭产品内事务。  
*   如果您的扩展收集信用卡信息，或者使用收集信用卡信息的第三方付款处理器，则付款处理必须符合当前的 PCI 数据安全标准 \ (PCI DSS\) 。  
    
#### 1.8.2 泄漏付费功能  

扩展和相关元数据必须提供有关所提供的产品内购买类型和价格范围的信息。  不得误导用户，并且必须清楚产品内促销和产品/服务的性质，包括任何试用体验的范围和条款。  如果扩展限制在试用期间或之后访问用户创建的内容，则必须提前通知用户。  此外，你的扩展必须让用户明确他们正在你的扩展中启动购买选项。  

### 1.9 通知  

你的扩展必须遵守通知的系统设置。  这意味着向用户显示广告和通知时，无论通知是否由 Microsoft 推送通知服务 \ (MPNS\) 、Windows 推送通知服务 \ (WNS\) 或其他任何服务提供，都必须与用户首选项一致。  如果用户在特定于产品或系统范围内禁用通知，扩展必须保持功能。  

如果你的产品使用 MPNS 或 WNS 传输通知，它必须符合以下要求：  

#### 1.9.1 一般指南  

通过 WNS 或 MPNS 提供的通知被视为产品内容，并受所有加载项目录开发人员策略限制。  

#### 1.9.2 通知的所有权  

不得掩盖或试图伪装从扩展启动的任何通知的来源。  

#### 1.9.3 无机密或敏感信息  

不得在通知中包括用户合理认为机密或敏感的任何信息。  

#### 1.9.4 通知的用途  

从扩展发送的通知必须与该扩展或你在 Microsoft Edge 加载项存储中发布的其他扩展相关，并且不得包含与扩展不相关的任何类型的促销邮件。  

### 1.10 广告行为准则和内容  

对于所有与广告相关联的活动，应符合以下要求：  

#### 1.10.1 用途  

扩展的主要内容不得是广告，并且广告必须与扩展中其他内容明确区分。  

#### 1.10.2 策略和协议  

你的扩展显示的任何广告内容必须遵守 [Microsoft 创意接受策略][MicrosoftAdvertisingCreativeAcceptancePolicies]。  
如果扩展显示广告，则显示的所有内容都必须符合应用开发人员协议和此策略[][MicrosoftAppDeveloperAgreement]的广告要求。  

#### 1.10.3 广告质量  

*   扩展的主要目的不得是让用户单击广告。  
*   你的扩展不得执行任何干扰或降低其显示的任何广告的可见性、价值或质量的任何功能。  

#### 1.10.4 促销  

如果你购买或创建促销广告市场活动以通过合作伙伴中心中的广告市场活动功能推广你的扩展，你向[][MicrosoftPartnerCenter]Microsoft 提供的所有广告材料（包括任何关联的登陆页面）都必须符合[Microsoft 创意][MicrosoftAdvertisingCreativeSpecifications]规范策略和[Microsoft 创意接受策略][MicrosoftAdvertisingCreativeAcceptancePolicies]。  

#### 1.10.5 通知用户Opt-Out广告Interest-Based通知  

你的隐私声明或使用条款必须让用户知道你计划向广告服务提供商发送个人信息，并且必须告知用户他们如何选择退出基于兴趣的广告。  

#### 1.10.6 其他准则  

如果你的分机面向 13 岁以下的儿童，如《儿童[在线隐私保护][FTCChildrensPrivacy]法》中的定义;你必须在合作伙伴中心通知 Microsoft[][MicrosoftPartnerCenter]这一事实，并确保你的扩展中显示的所有广告内容都适用于 13 岁以下的儿童。  

## 2 内容策略  

以下策略适用于内容和元数据 \ (包括发布者名称、扩展名、扩展图标、扩展说明、扩展屏幕截图、扩展预告片和预告片缩略图，以及 Microsoft Edge 加载项中提供用于分发的其他任何扩展元数据\) 。  内容是指扩展中包含的图像、声音、视频和文本、通过扩展公开的磁贴、通知、错误消息或广告，以及从服务器或扩展连接到的任何内容。  由于世界各地都使用扩展和 Microsoft Edge 加载项，因此这些要求在地区和文化规范的上下文中进行解释和应用。  

### 2.1 Microsoft Edge 加载项目录一览的内容要求  

提交到扩展的元数据和其他内容可能不包含成熟内容。  
不符合 Microsoft Edge 加载项应用商店一览要求的提交将被拒绝或立即删除。  

### 2.2 内容，包括名称、徽标、原始和第三方  

扩展和相关元数据中所有内容都必须最初由您创建，或由第三方权利持有者适当许可，并且必须仅在权限持有者允许的情况下或法律允许的情况下使用。  

### 2.3 危害风险  

#### 2.3.1 要求  

扩展不得包含任何促进或宣传以下实际活动的内容：\ (\) 极端或无端暴力;\ (b\) 违反;\ (c\) 非法武器;或 \ (d\) 对人员、动物或真实或个人属性使用武器。  

#### 2.3.2 责任  

你的扩展不得：\ (\) 给最终用户或任何其他人员或动物带来安全风险，也不得对最终用户或其他任何人或动物造成危害、损害或其他任何损害;或 \ (b\) 对真实或个人属性带来风险或造成损坏。  你单独负责所有扩展安全测试、证书获取以及任何相应功能安全措施的实施。  不得禁用任何平台安全或舒适功能，并且必须在扩展中包括所有适用的法律要求和行业标准警告、通知和免责声明。  

### 2.4 破坏性、中性、Slandatory 和威胁  

您的扩展不得包含任何具有破坏性、中性、中性或威胁性的内容。  

### 2.5 冒犯性内容  

扩展和相关元数据不得包含潜在敏感或冒犯性内容。  由于当地法律或文化规范，某些国家/地区的内容可能被视为敏感内容或冒犯性内容。  此外，你的扩展和相关元数据不得包含基于种族、族裔、国家/地区、语言、性别、年龄、残障、精神、性取向、年长身份或任何其他社会组的成员身份而宣传种族、宗教或暴力的内容。  

### 2.6 滥用、滥用和滥用  

你的扩展不得包含任何促进或宣传滥用或滥用商品、非法产品或非法内容的内容。  

### 2.7 成人内容  

你的扩展不得包含或显示合理人员认为具有黄色或暗示性的内容。  

### 2.8 禁止的内容、服务和活动  

您的扩展必须遵守以下条件。  

*   你的扩展不得包含内容或提供促进在线购物的服务。  在线购物包括但不限于在线信用卡、体育俱乐部、彩金或提供现金或其他价值的游戏。  
*   您的扩展不得提供对网站内容的未经授权的访问，例如绕过付费区或登录限制。  
*   您的扩展不得提供、鼓励或启用未经授权的访问、下载或流式传输受版权保护的内容或媒体。  
*   您的扩展不得挖掘加密货币。  
    
### 2.9 非法活动  

你的扩展不得包含鼓励、促进或鼓励现实世界中的非法活动的内容或功能。  

### 2.10 过度亵渎和不当内容  

*   您的扩展不得包含过多或无理的亵渎。  
*   您的扩展不得包含或显示合理用户认为具有攻击性的内容。  
    
### 2.11 国家/地区特定要求  

不允许在扩展所针对的任一国家/地区中冒犯性内容。  由于当地法律或文化规范，一些内容可能会在某些国家/地区视为冒犯性内容。  在某些国家/地区可能被视为具有冒犯性的内容示例包括以下各项：  

**中国**  

*   违禁的色情内容  
*   有争议的领土或区域引用  
*   提供或允许对违反适用的当地法律的内容或服务的访问  
    
### 2.12 年龄分级  

#### 2.12.1 成熟内容  

将扩展提交到 [合作伙伴][MicrosoftPartnerCenter]中心时，必须指示扩展是否显示应标记为"成熟"的内容。  在确定扩展的分级时，请考虑应用中的所有内容（包括用户生成的内容和广告）以及你的扩展链接的内容。  如果你指示你的扩展不包含任何"成熟"内容，你负责保持此分级的准确性。  无论为扩展提供何种分级，它仍然必须遵守 Microsoft Edge 加载项开发人员策略的所有内容要求  

#### 2.12.2 分级更改  

如果您的扩展提供的内容 \ (如用户生成的内容、零售内容或其他基于 Web 的内容\) 可能适用于比分配分级更高的年龄分级，则必须要求用户选择通过使用内容筛选器或使用预先存在的帐户登录来接收此类内容。  

### 2.13 视频  

如果你在一览中提交促销视频，它应遵循此策略中提到的所有内容指南。  如果选择提供 YouTube 链接，必须确保对要嵌入的特定视频禁用广告。  有关如何在 YouTube 上启用和禁用广告的信息，请参阅[support.google.com/youtube/answer/2531367?ref_topic=7072227和][GoogleYoutubeAnswer2531367Topic7072227][support.google.com/youtube/answer/132596。][GoogleYoutubeAnswer132596]
