# Microsoft Edge Addons 目录开发人员策略  

## 本文档的简介和目标  

感谢你关注开发 Microsoft Edge Addons 目录的扩展。  Microsoft Edge Addons 目录开发人员策略 \ （Addons Catalog 开发人员策略 \）适用于你的扩展，包括通过[合作伙伴中心][MicrosoftPartnerCenter]提交扩展以及通过 Microsoft Edge Addons 提供此类扩展。  

## 原则  

下面是一些入门准则：  

*   你应该在 Microsoft Edge 的扩展中提供唯一且独特的值。  提供从 Microsoft Edge Addons 目录下载扩展的有说服力的理由 \ （Microsoft Edge Addons \）。  
*   您不能误导我们的联合用户对您的分机内容、提供服务的人，等等。  
*   您不得尝试给用户、系统或生态系统的用户。  Microsoft Edge Addons 中不存在任何类型欺诈的地方;成为 it 评级和评论操作、信用卡欺诈或其他欺诈活动。  

遵守这些 Addons 目录开发人员策略应有助于做出更好的选择来增强您的扩展的吸引力和受众。  

您的扩展对成百上千个用户的体验非常重要。  我们期待着您创建和欣慰的内容，帮助您向世界提供您的分机。  

## 1. 产品策略  

### 1.1 Distinct 函数 & 值;精确表示形式  

你的扩展和关联的元数据必须准确、清楚地反映你描述的源、功能和功能。  

#### 1.1.1 扩展必须具有单个用途  

您的扩展必须具有一个具有狭窄功能的单一用途。  

#### 1.1.2 介绍您的扩展  

扩展的所有方面都应准确描述扩展的功能、特性和任何重要限制，包括必需或受支持的输入设备。  在首次运行体验期间，您的扩展的价值主张必须明确。  您的分机号码不能使用与其他分机号类似的名称或图标，如果您不具有进行该表示法的权限，则不得声明表示公司、政府正文或其他实体。  

#### 1.1.3 功能  

您的扩展必须完全有效。  

#### 1.1.4 搜索和发现  

搜索词不得超过七个唯一术语，并且应该与您的分机相关。  

#### 1.1.5 提供相应的详细信息  

你必须提供有关你的扩展的独特和信息性详细信息，以及为你的扩展列出 \ （元数据 \）的功能。  您的扩展必须提供有价值且优质的用户体验。  你的扩展还必须在 Microsoft Edge Addons 中具有活动状态。  

#### 1.1.6 稳定性和性能  

你的扩展不得对 Microsoft Edge 的性能或稳定性产生负面影响。  

#### 1.1.7 混淆  

不允许使用带有模糊代码的扩展。  这包括扩展包中的代码以及从 web 提取的任何外部代码或资源。  如果代码未 reviewable，可能会要求你重构代码的部分内容。  

#### 1.1.8 更改浏览器设置  

您的分机**不能、未经适当的用户同意**、更改或看起来改变、浏览器功能或设置，包括但不限于：地址栏搜索提供程序和建议、起始页或主页、新选项卡页以及添加或删除收藏夹。  

对浏览器设置所做的任何更改都应明确记录在扩展的描述中。  

你的扩展可能只会修改密钥设置，以便将 Microsoft 网页或服务替换为第三方的 Microsoft 网页或服务（例如，需要使用第三方搜索引擎，或者将主页设置为第三方 web 属性 \）（如果你使用的是此类第三方或其他相关联）。  

### 1.2 安全性  

您的扩展不得危害或危害用户安全，或设备、系统或相关系统的安全或功能。  

#### 1.2.1 内容安全策略  

> [!NOTE]
> 如果你对扩展的更改超出了所述功能，则对代码所做的任何更改都必须符合[Microsoft Edge 内容安全策略][MicrosoftEdgeContentSecurityPolicyRemoteScript]。  示例：你的扩展不应下载远程脚本，随后以与所述功能不一致的方式运行该脚本。  

#### 1.2.2 有害和恶意软件  

您的扩展不得包含或启用恶意软件，由 Microsoft[不受欢迎和恶意软件][MicrosoftIdentifiesMalwareUnwantedApplications]的标准定义。  

#### 1.2.3 其他软件的相关性  

你的扩展可能依赖于非集成软件 \ （如另一产品、模块或服务）来提供主要功能，前提是你在描述中公开了依赖关系  

#### 1.2.4 扩展更新  

除非 Microsoft 允许，否则你的扩展必须仅通过 Microsoft Edge Addons 更新。  

### 1.3 产品可测试  
扩展必须可测试。  如果不能出于任何原因测试您的扩展，包括但不限于以下项目，则您的扩展可能无法实现此要求。  

#### 1.3.1 用户凭据  
如果您的扩展需要登录凭据，请使用 "**证书说明**" 字段提供工作的演示帐户。  

#### 1.3.2 服务的可用性  

如果您的扩展需要访问服务器，则服务器必须能够正常工作以验证其是否正常工作。  

### 1.4 可用性  

您的分机必须满足扩展存储标准的可用性，包括（但不限于）以下各小节所列的扩展存储标准。  

#### 1.4.1 跨平台的兼容性  

扩展应与 Microsoft Edge 兼容，这些设备和平台上的应用都可以下载。  如果某个扩展在与它不兼容的设备上下载，应在启动时检测到该扩展，并向用户显示一条消息，详细介绍设备必须满足的要求，以便与你的扩展兼容。  

#### 1.4.2 用户体验  

扩展必须立即启动，并且必须保持对用户输入的响应。  扩展必须继续运行并保持对用户输入的响应。  扩展必须正常关闭，并且不会意外关闭。  在处理异常之后，扩展应处理异常并保持对用户输入的响应。  

### 1.5 个人信息  

以下要求适用于访问个人信息的扩展。  个人信息包括标识或可用于标识人员或与此类信息或数据相关联的所有信息或数据。  

#### 1.5.1 仅在必要时收集个人信息  

您的扩展可能会收集、访问、使用或传输个人信息 \ （包括 web 浏览活动 \）;仅在需要时，并且仅在具有突出披露的、面向用户的功能中使用。  

#### 1.5.2 维护隐私策略  

无论您的分机是访问、收集还是传输个人信息;如果法律要求，您必须提供与您的隐私策略相符的通知。  您的隐私策略必须通知用户通过您的扩展访问、收集或传输的个人信息，如何使用、存储和保护这些信息，以及如何使用该信息来指示其泄漏方的类型。  您的隐私策略必须描述用户对其信息的使用和共享的控制，以及他们访问其信息的方式，并且必须遵守适用的法律和法规。  将新功能和功能添加到你的扩展时，你的隐私策略必须保持最新状态。  

如果你向 Microsoft 提供你的隐私策略，则表示你同意允许 Microsoft 与你的扩展用户共享此类隐私策略。  

#### 1.5.3 与第三方共享数据  

仅在从这些用户获得自愿同意后，您才可以通过您的扩展或关联的元数据将扩展的用户的个人信息发布到外部服务或相关的元数据。  自愿参加同意意味着用户在你的扩展的用户界面中为请求的活动提供快速权限，然后执行以下操作：  

*   向你的用户介绍如何访问、使用或共享信息，以及如何将信息公开给哪些参与方的类型以及  
*   向用户提供扩展用户界面中的一种机制，用户可在此界面中选择权限并选择退出 rescind。  

#### 1.5.4 非用户共享信息  

如果您通过您的扩展或元数据将某人的个人信息发布到外部服务或第三方，但其信息的共享用户不是您的扩展的用户;  

1.  您必须获得书面同意才能发布个人信息。  
1.  您必须允许共享其信息的人员在任何时间撤消同意。  
1.  您的隐私政策必须明确公开，您可以通过这种方式收集个人信息。  
1.  如果适用法律要求，您必须删除任何个人的个人信息，包括以这种方式收集的个人信息。  
1.  如果你的扩展向用户提供对其他人的个人信息的访问权限，则此要求也适用。  

#### 1.5.5 信息安全地传输信息  

如果您的扩展收集、存储或传输个人信息，它必须使用新式加密方法安全地执行此操作。  

#### 1.5.6 高度敏感的信息  

您的扩展不得收集、存储或传输高度敏感的个人信息（如健康或财务数据），除非信息与您的扩展功能相关。  在收集、存储或传输此类信息之前，您的分机还必须获得快速用户同意。  

### 1.6 权限  

你的扩展必须仅请求具有正常运行所必需的权限。  你必须提供有关扩展的工作方式的说明。  您的扩展必须仅按所述执行。  你的扩展可能不会请求超越所需的功能，而不是所声明的执行和函数所需的功能。  

### 1.7 本地化  

你应该本地化扩展声明支持的所有语言的扩展。  应将你的扩展说明的文本本地化为你声明的每种语言。  
如果你的扩展已本地化，并且某些功能在本地化版本中不可用，则必须在 "扩展说明" 中明确声明或显示本地化限制。 扩展提供的体验必须非常类似于它支持的所有语言。  

### 1.8 金融交易  

如果你的产品包含产品内购买、套餐、虚拟货币、计费功能或捕获财务信息，则为以下各节中的要求适用。  

#### 1.8.1 付费功能  

你的扩展可使用户能够使用通过第三方购买机制或 API 购买的数字内容或服务。  

您必须使用安全的第三方购买 API 来购买实物商品或服务。  您必须使用安全的第三方购买 API，以便与任何其他服务（包括真实的赌博或慈善捐赠）进行付款。  

*   如果您的分机便于或收集慈善捐赠或开展促销抽奖活动或比赛，您必须遵守适用法律。  
*   您还必须明确声明，Microsoft 不是此次促销活动的集资者或主办方。  
*   您的分机中销售的产品内产品不能转换为任何合法的有效货币 \ （如 USD、欧元等）或任何实物商品或服务。  

以下要求适用于您使用安全的第三方购买 API：  

*   在交易记录期间或从用户处收集任何付款或财务信息时;您的扩展必须标识商务事务提供商，验证用户身份，并获取有关交易的用户确认。  商务事务提供商维护用于财务交换的安全平台。  
*   你的扩展可能会向用户提供保存此身份验证的功能，但用户必须能够对每个事务都要求身份验证，或者关闭产品内事务。  
*   如果您的扩展收集了信用卡信息或使用的第三方付款处理器收集信用卡信息，则付款处理必须满足当前的 PCI 数据安全标准 \ （PCI DSS \）。  

#### 1.8.2 公开付费功能  

您的分机和相关元数据必须提供有关产品内购买类型以及价格范围的信息。  您不得误导用户，并且必须明确了解产品内促销的性质以及任何试用体验的范围和条款。  如果你的扩展在试用期间或之后限制对用户创建的内容的访问，则必须提前通知用户。  此外，你的扩展必须让用户清楚地在扩展中启动购买选项。  

### 1.9 通知  

您的扩展必须遵守通知的系统设置。  这意味着，无论通知是否由 Microsoft 推送通知服务（MPNS \）、Windows 推送通知服务（WNS \）还是任何其他服务提供，用户的广告和通知的任何演示文稿都必须与用户首选项保持一致。  如果用户在特定于产品或系统范围的基础上禁用通知，则你的扩展必须保持正常运行。  

如果你的产品使用 MPNS 或 WNS 传输通知，则它必须符合以下要求：  

#### 1.9.1 常规指南  

通过 WNS 或 MPNS 提供的通知被视为产品内容，并且受限于所有 Addons 目录开发人员策略。  

#### 1.9.2 通知的所有权  

您不得遮盖或尝试伪装从您的扩展发起的任何通知的来源。  

#### 1.9.3 没有机密或敏感信息  

您不得在通知中包含任何信息用户可能会合理地考虑保密或敏感信息。  

#### 通知的1.9.4 用途  

从你的扩展发送的通知必须与该扩展名或你在 Microsoft Edge Addons 目录中发布的其他扩展项相关，并且不能包含与你的扩展无关的任何类型的促销消息。  

### 1.10 广告行为和内容  

对于所有与广告相关联的活动，应符合以下要求：  

#### 1.10.1 用途  

您的扩展的主要内容不能广告，广告必须清楚地与您的分机中的其他内容有区别。  

#### 1.10.2 政策和协议  

您的扩展显示的任何广告内容都必须遵守[Microsoft 创造性接受政策][MicrosoftAdvertisingCreativeAcceptancePolicies]。  
如果你的扩展显示广告，则显示的所有内容必须符合[应用开发人员协议][MicrosoftAppDeveloperAgreement]和此策略的广告要求。  

#### 广告的1.10.3 质量  

*   您的分机的主要用途必须不是让用户单击广告。  
*   您的扩展不得执行任何干扰或降低其所显示广告的可见性、价值或质量的任何操作。  

#### 1.10.4 促销  

如果购买或创建促销广告市场活动以通过[合作伙伴中心][MicrosoftPartnerCenter]中的广告市场活动功能推广您的扩展，则您提供给 microsoft 的所有广告材料（包括任何相关登陆页面）都必须遵守[Microsoft 创意规范政策][MicrosoftAdvertisingCreativeSpecifications]和[microsoft 创意接受政策][MicrosoftAdvertisingCreativeAcceptancePolicies]。  

#### 1.10.5 通知用户选择退出基于兴趣的广告  

您的隐私声明或使用条款必须让用户知道您计划向广告服务提供商发送个人信息，并且必须告诉用户他们可能选择退出基于兴趣的广告。  

#### 1.10.6 其他指南  

如果你的分机由13岁以下的儿童所有，如[儿童在线隐私保护法案][FTCChildrensPrivacy]中所定义;您必须在 "[合作伙伴中心][MicrosoftPartnerCenter]" 通知 Microsoft 此事实，并确保您的 "扩展" 中显示的所有广告内容适用于13岁以下的儿童。  

## 2个内容策略  

以下策略适用于内容和元数据 \ （包括发布者名称、扩展名、扩展名图标、扩展说明、扩展屏幕截图、扩展尾端和尾端缩略图以及任何其他扩展元数据 \），用于在 Microsoft Edge Addons 中分发。  内容意味着在扩展中包含的图像、声音、视频和文本、通过扩展公开的磁贴、通知、错误消息或广告，以及从服务器或扩展连接到的任何内容。  由于全球各地使用的是扩展和 Microsoft Edge Addons，因此这些要求在区域和文化规范的上下文中进行解释和应用。  

### 2.1 Microsoft Edge 加载项目录列表的内容要求  

与您的扩展一起提交的元数据和其他内容可能不包含成人内容。  
不满足扩展存储列表要求的提交将被拒绝或立即删除。  

### 2.2 内容，包括名称、徽标、原始和第三方  

你的扩展和关联的元数据中的所有内容都必须是你或从第三方权限持有人处获得的，并且必须仅由权限持有者允许或在法律允许的情况中使用。  

### 2.3 危害风险  

#### 2.3.1 要求  

您的分机不得包含任何有助于或 glamorizes 以下现实活动的内容： \ （a \）极端或免费暴力;\ （b）人工权限违规;\ （c）创建非法武器;或 \ （d）针对个人、动物或个人财产使用武器。  

#### 2.3.2 责任  

您的分机号不得： \ （a）为最终用户或任何其他人或动物带来安全风险，也不会造成 discomfort、人身伤害或任何其他损害。或者 \ （b）承担或导致真正或个人财产的损失。  您对所有扩展安全测试、证书购买和实施任何适当的功能安全措施承担全部责任。  您不得禁用任何平台安全功能或舒适功能，并且您必须在您的分机中包括所有适用的法律要求和行业标准的警告、通知和免责声明。  

### 2.4 诽谤性、Libelous、Slanderous 和威胁  

你的扩展不得包含任何为诽谤性、libelous、slanderous 或威胁的内容。  

### 2.5 冒犯性内容  

您的扩展和关联的元数据不能包含潜在敏感内容或攻击性内容。  由于当地法律或文化规范，内容可能被视为在某些国家/地区的敏感或攻击性。  此外，您的扩展和关联的元数据不得包含基于种族、ethnicity、民族、语言、性别、年龄、伤残、宗教、色情方向、hatred 或任何其他社交团体的成员的内容。  

### 2.6 喝酒、抽烟和毒品  

您的分机不得包含任何有助于或 glamorizes 的内容，或 irresponsible 使用喝酒或抽烟的产品或毒品。  

### 2.7 成人内容  

您的分机不得包含或显示合理的人认为色情或色情的内容。  

### 2.8 禁止的内容、服务和活动  

您的分机号必须遵守下列条件。  

*   您的扩展不得包含内容或提供便于联机赌博的服务。  在线赌博包括但不限于在线 casinos、体育打赌、lotteries 或提供现金或其他价值奖品的技术游戏。  
*   您的扩展不得向网站内容提供未经授权的访问权限，例如避开 paywalls 或登录限制。  
*   你的扩展不得提供、鼓励或启用受版权保护的内容或媒体的未经授权访问、下载或流式处理。  
*   您的分机不一定是 cryptocurrency。  

### 2.9 非法活动  

你的扩展不得包含鼓励、促进或 glamorizes 在现实世界中非法活动的内容或功能。  

### 2.10 过高的猥亵和不当内容  

*   您的分机不得包含多余或不必要的猥亵语言。  
*   您的扩展不得包含或显示合理的人认为讨厌的内容。  

### 2.11 国家/地区特定要求  

不允许在你的分机所在的任何国家/地区有冒犯性的内容。  由于当地法律或文化规范，一些内容可能会在某些国家/地区视为冒犯性内容。  在某些国家/地区可能被视为具有冒犯性的内容示例包括以下各项：  

**中国**  

*   违禁的色情内容  
*   有争议的领土或区域引用  
*   提供或允许对违反适用的当地法律的内容或服务的访问  

### 2.12 年龄分级  

#### 2.12.1 成人内容  

向[合作伙伴中心][MicrosoftPartnerCenter]提交扩展时，必须指明您的扩展是否显示应标记为 "成人" 的内容。  确定你的扩展的分级时，请考虑你的应用中的所有内容，包括用户生成的内容和广告，以及你的扩展链接的内容。  如果你指示你的扩展不包含任何 "成人" 内容，你将负责保持此评级的准确性。  无论为您的扩展提供的评分，它仍然必须遵守 Microsoft Edge Addons 开发者策略的所有内容要求  

#### 2.12.2 分级更改  

如果你的扩展提供的内容 \ （如用户生成的、零售版或其他基于 web 的内容 \）可能适用于比分配的分级更高的年龄分级，则你必须要求用户选择通过使用内容筛选器或使用预先存在的帐户登录来接收此类内容。  

### 2.13 视频  

如果你在列表中提交促销视频，则应遵循本政策中提及的所有内容指南。  如果你选择提供 YouTube 链接，则必须确保已针对要嵌入的特定视频禁用广告。  有关 YouTube 上启用和禁用广告的详细信息，请参阅[support.google.com/youtube/answer/2531367?ref_topic=7072227][GoogleYoutubeAnswer2531367Topic7072227]和[support.google.com/youtube/answer/132596][GoogleYoutubeAnswer132596]。
